# 🧱 Modelagem no Tinkercad — Sistema IoT com ESP32

## 📌 Visão Geral

Este documento apresenta a utilização do Tinkercad no projeto Sistema IoT de Monitoramento Ambiental com ESP32.

O Tinkercad foi utilizado para representar visualmente a organização física do sistema, incluindo o ESP32, sensores, atuadores e estrutura do gabinete.

Essa etapa contribui para demonstrar como os componentes poderiam ser organizados em uma aplicação física real.

---

## 🎯 Objetivo da Modelagem

O objetivo da modelagem no Tinkercad é criar uma representação visual do sistema, facilitando a compreensão da disposição dos componentes e da estrutura física do projeto.

A modelagem permite visualizar:

* Organização do ESP32
* Posicionamento dos sensores
* Posicionamento dos atuadores
* Estrutura do gabinete
* Distribuição dos componentes
* Possível montagem física do protótipo

---

## 🧩 Componentes Representados

Na modelagem do projeto, podem ser representados os seguintes componentes:

* ESP32
* Sensores ambientais
* LED de sinalização
* Buzzer de alerta
* Relé
* Fios de conexão
* Estrutura do gabinete
* Área de ventilação
* Espaço para alimentação elétrica
* Área para passagem de cabos

---

## 🏗️ Estrutura do Gabinete

O gabinete tem como objetivo proteger e organizar os componentes eletrônicos do sistema.

Características desejadas:

* Espaço interno para o ESP32
* Aberturas para sensores
* Aberturas para ventilação
* Espaço para passagem de cabos
* Fixação dos componentes
* Organização visual do protótipo
* Proteção dos circuitos

---

## 🔧 Importância do Gabinete

O gabinete é importante porque permite que o projeto deixe de ser apenas uma simulação e se aproxime de uma aplicação real.

Ele contribui para:

* Melhor organização dos componentes
* Proteção do circuito
* Facilidade de transporte
* Melhor apresentação do projeto
* Visualização da solução física
* Maior profissionalismo na demonstração

---

## 🔄 Relação com o Projeto IoT

A modelagem no Tinkercad complementa a simulação no Wokwi.

Enquanto o Wokwi é utilizado para testar a lógica, sensores e funcionamento do circuito, o Tinkercad ajuda a representar a parte física e estrutural do sistema.

Dessa forma, o projeto apresenta uma visão mais completa, envolvendo:

* Software
* Hardware
* Simulação
* Estrutura física
* Monitoramento em nuvem
* Automação

---

## 📊 Possível Organização Física

| Componente                      | Localização sugerida              |
| ------------------------------- | --------------------------------- |
| ESP32                           | Parte central do gabinete         |
| Sensor de temperatura e umidade | Área externa ou lateral           |
| Sensor de luminosidade          | Área superior ou frontal          |
| Sensor de qualidade do ar       | Área com abertura para ventilação |
| LED                             | Parte frontal do gabinete         |
| Buzzer                          | Parte interna com saída de som    |
| Relé                            | Área isolada para acionamento     |
| Cabos                           | Passagem lateral ou inferior      |

---

## ✅ Resultado Esperado

Com a modelagem no Tinkercad, espera-se demonstrar como o sistema poderia ser montado fisicamente, permitindo melhor compreensão do projeto e facilitando futuras implementações reais.

A modelagem também valoriza a apresentação do projeto, mostrando preocupação com organização, estrutura, acabamento e aplicação prática.
