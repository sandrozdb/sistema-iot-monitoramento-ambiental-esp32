# 📊 Resultados Esperados — Sistema IoT de Monitoramento Ambiental com ESP32

## 📌 Visão Geral

Este documento apresenta os principais resultados esperados com o desenvolvimento do Sistema IoT de Monitoramento Ambiental com ESP32.

O projeto tem como objetivo demonstrar a aplicação prática de Internet das Coisas, sensores, sistemas embarcados, comunicação Wi-Fi e monitoramento remoto em nuvem.

---

## 🎯 Objetivo dos Resultados

O objetivo principal é validar o funcionamento de uma solução capaz de coletar dados ambientais em tempo real, processar essas informações no ESP32, acionar alertas quando necessário e enviar os dados para uma plataforma em nuvem.

---

## ✅ Resultados Funcionais Esperados

Ao final do projeto, espera-se que o sistema consiga:

* Realizar leitura de sensores ambientais
* Medir temperatura do ambiente
* Medir umidade do ar
* Ler níveis de luminosidade
* Monitorar umidade do solo
* Simular ou medir qualidade do ar
* Processar os dados coletados no ESP32
* Identificar valores fora dos limites definidos
* Acionar LED, buzzer ou relé em condição de alerta
* Enviar dados para o ThingSpeak
* Permitir acompanhamento remoto das informações
* Exibir histórico das medições em gráficos

## 🧾 Comparação entre Monitoramento Manual e Monitoramento IoT

| Critério                | Monitoramento Manual          | Monitoramento com IoT    |
| ----------------------- | ----------------------------- | ------------------------ |
| Coleta de dados         | Manual                        | Automática               |
| Frequência de medição   | Limitada                      | Contínua                 |
| Visualização remota     | Não disponível                | Disponível em nuvem      |
| Registro histórico      | Manual ou inexistente         | Armazenado na plataforma |
| Detecção de alerta      | Depende de verificação humana | Automática               |
| Escalabilidade          | Baixa                         | Alta                     |
| Integração com sistemas | Limitada                      | Possível                 |
| Análise de dados        | Mais difícil                  | Facilitada por gráficos  |

---

## 🚨 Resultados em Condição de Alerta

O sistema deve identificar situações fora dos limites definidos, como:

* Temperatura elevada
* Umidade do ar baixa
* Umidade do solo insuficiente
* Qualidade do ar fora do padrão
* Variações anormais em sensores analógicos

Quando essas condições forem detectadas, o sistema pode acionar:

* LED para alerta visual
* Buzzer para alerta sonoro
* Relé para representar acionamento de dispositivo externo

---

## ☁️ Resultados no ThingSpeak

Com a integração em nuvem, espera-se que o ThingSpeak consiga exibir:

* Gráfico de temperatura
* Gráfico de umidade do ar
* Gráfico de luminosidade
* Gráfico de umidade do solo
* Gráfico de qualidade do ar
* Gráfico da variável analógica simulada

Esses dados permitem acompanhar o comportamento ambiental ao longo do tempo e analisar possíveis padrões.

## ✅ Conclusão

O projeto demonstra como uma solução de baixo custo baseada em ESP32 pode ser utilizada para monitoramento ambiental em tempo real.

A integração entre sensores, atuadores, microcontrolador e plataforma em nuvem permite criar uma solução funcional, escalável e aplicável a diferentes cenários, como agricultura, automação residencial, cidades inteligentes e monitoramento ambiental.
